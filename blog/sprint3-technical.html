Control Flow and Loops

Here are some examples of statements and loops

If/Else Statement

 

if(ice-cream == 'delicious') {
    console.log('Ice cream is delicious')
}  else {
    console.log('Error, Ice cream is delicious')
}

Switch Statement

 

switch (ice-cream) {
    case 'chocolate':
        eatIceCream()
        break;
    case 'raisin':
        binIceCream()
        break;
}

For Loop

 

for (let i = 0; i < iceCream.length; i++) {
    iceCreamInTum += 1
}

Do While Loop

 

let i = 0
do {
    eatIceCream()
    i++
} while (i < iceCream.length)

To control the flow of the code is to direct it in a particular directions using different checks. By using an if/else statement or a switch statement, you can check to see if certain conditions are true, and execute different code blocks based on the result. In the switch statement above for example, we are checking the flavour of the ice cream and executing an action based on the result - eating the ice cream if it is chocolate, and throwing it out if it is raisin. Raisins should be illegal.

Loops are used to repeat a block of code until a condition becomes true. In the above Do While Loop, we eat ice cream then continue to do so until there is no ice cream left.

The DOM

The DOM, or Document Object Model is a representation of all the objects that come together to form a webpage. It takes the form of a tree-like structure, starting with the window at the top (through which the user views the site e.g. firefox), then down to the document (the site itself). From that is all the other elements like images and forms and anchors and the list goes on and on and on.

Everything you see (and don't see) on a website will be represented somewhere in the DOM.

The beauty of this is you can use JavaScript (and other languages) to access the dom and programmatically change it. You could get a reference to a header and change it's colour. You could change the dimensions of an image. You could even use it to insert totally new elements, or make existing ones disappear. It's a very powerful tool. Use it wisely.

Arrays and Objects - Accessing Data

Here is an example of an array and an object

Array

 

iceCream['Chocolate', 'Vanilla', 'Feijoa', 'Orange Choc-chip']

Object

 

iceCream = {
    flavour: 'chocolate',
    temperature: '6',
    cone: true
}

An array is essentially a list. We have a list of ice cream flavours contained in the variable 'ice cream'. Each item in that list has an index assigned to it, starting at 0 for the item in the first position. To access a particular item in that list we can use its index. For example to access feijoa, we would use iceCream[2]. Note the use of square brackets here. There are various other methods that can be used on an array, such as iceCream.length which would return the number of items on the list, and iceCream.push('Mint'), which would add Mint to the list.

An object is similar to an array in that it is like a list, but instead of being a list of items, it is a list of key value pairs relating to a singular item. In the example we have the object iceCream. It has several key parameters, these being flavour, temperature, and whether or not it comes in a cone. Each of these keys has a value assigned to it. To access the values we can use dot notation. For instance, iceCream.temperature will return 6.

To further complicate things, it is possible to have an array of objects, or to have an array within your object, or both! Then you need to use a combination of dot notation and square bracket notation to access the correct field.

What are functions and why are they useful?

'Function' is another word for a method. It is a set of instructions to complete a specific task. A class consists of many different functions working together. A function could look like this example on the right. First we have the function declaration and the name of the function (function buyIceCream) which takes a parameter called money. The function then performs a check - if the value of money is greater than 4.5, it returns iceCream. If not, it returns sadness.

Functions are useful because we can reuse them as much as we like. These steps might need to be used multiple times through our code, and by calling a function, it means we don't have to write the same code out each time. 


function buyIceCream(money) {
    if(money > 4.5) {
        return iceCream
    } else {
        return sadness
    }
}
